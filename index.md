## ★★★★★ ゲームチェンジャー（13件）

### エージェント・マルチエージェント

#### 1. エージェントチーム機能（リサーチプレビュー）
- **バージョン:** 2.1.32
- **カテゴリ:** Agent/Multiagent
- **概要:** 複数の Claude Code エージェントがチームとして協調動作する機能（`CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` で有効化）。単一エージェントのコンテキスト長・並列処理の限界を突破し、大規模・複雑なタスクを分散処理できる。将来の業界標準となりうるアーキテクチャレベルの変革。

#### 2. `--worktree`（`-w`）フラグによる隔離 git worktree 起動
- **バージョン:** 2.1.49
- **カテゴリ:** Agent/Multiagent
- **概要:** Claude を独立した git worktree 内で起動し、メインブランチへの影響を隔離しながら作業できる。複数エージェントが同一リポジトリで衝突せずに同時作業でき、エージェントチーム機能と組み合わせることで真価を発揮する。

#### 3. 新タスク管理システム（依存関係トラッキング）
- **バージョン:** 2.1.16
- **カテゴリ:** Agent/Multiagent
- **概要:** 単純なタスクリストから「依存関係を持つワークフローグラフ」へ進化。「DBスキーマ作成 → API実装 → テスト作成 → ドキュメント生成」のような依存連鎖を自律管理でき、エージェントオーケストレーションの品質を根本的に引き上げる。

### 設定・エンタープライズ

#### 4. Bash パーミッションルールのワイルドカードサポート
- **バージョン:** 2.1.0
- **カテゴリ:** Config/Enterprise
- **概要:** `Bash(npm *)` や `Bash(git * main)` のようにワイルドカード `*` でコマンドファミリーを一括承認可能。セキュリティを保ちながら承認ダイアログの手間を大幅削減。CI/CD環境やチーム共有設定での運用効率が根本的に変わる。

#### 5. modelOverrides 設定
- **バージョン:** 2.1.73
- **カテゴリ:** Config/Enterprise
- **概要:** モデルピッカーのエントリをAWS Bedrock推論プロファイルARN等のカスタムプロバイダーモデルIDにマッピング可能。ベンダーロックインを避けつつ、データローカリティや監査といったガバナンス要件を満たしながら Claude Code を活用できる。

#### 6. Opus 4.6 1M コンテキストウィンドウのデフォルト有効化
- **バージョン:** 2.1.75
- **カテゴリ:** Config/Enterprise
- **概要:** Max・Team・Enterpriseプランで100万トークンのコンテキストウィンドウが標準有効化。数十万行規模のモノレポや数百ページの仕様書を単一セッションで扱えるようになり、コンテキスト喪失によるミスや繰り返し説明のコストを大幅削減。

### DX・UI

#### 7. カスタマイズ可能なキーボードショートカット
- **バージョン:** 2.1.18
- **カテゴリ:** DX/UI
- **概要:** `/keybindings` コマンドでコンテキスト別のキーバインドを設定可能。Vim/Emacs/VS Code のキーバインドに合わせることでコンテキストスイッチのストレスを解消。一度設定すれば全セッションで恩恵を受け続けるため投資対効果が極めて高い。

### フック・拡張性

#### 8. PreToolUse フックの additionalContext 返却
- **バージョン:** 2.1.9
- **カテゴリ:** Hooks/Extensibility
- **概要:** `PreToolUse` フックがモデルへ `additionalContext` を返却可能に。フックが「監視・ブロック」を超え、モデルの判断に情報を提供する能動的な存在に昇格。ファイル編集前に設計ドキュメントを自動注入したり、bash 実行前に環境情報を追加したりできる。

#### 9. PreToolUse / PostToolUse / Stop フック（エージェント・スキルフロントマターへの拡張）
- **バージョン:** 2.1.0
- **カテゴリ:** Hooks/Extensibility
- **概要:** エージェント定義およびスキルのフロントマターで `PreToolUse`、`PostToolUse`、`Stop` フックをサポート。グローバル設定に頼らずエージェント・スキル単位でツール使用を制御でき、マルチエージェント構成での安全性と柔軟性が劇的に向上。

#### 10. HTTP フック
- **バージョン:** 2.1.63
- **カテゴリ:** Hooks/Extensibility
- **概要:** フックのアクションとして URL に JSON を POST し、レスポンスの JSON を受け取る HTTP フック。シェルスクリプトを介さずに Webhook・REST API・社内サービスと直接連携でき、エンタープライズ環境でのセキュリティ審査・承認プロセスの自動化が容易に。

### MCP統合

#### 11. MCP elicitation サポート
- **バージョン:** 2.1.76
- **カテゴリ:** MCP Integration
- **概要:** MCPサーバーがタスク実行中にインタラクティブダイアログでユーザーへ構造化入力を要求可能に。`Elicitation` / `ElicitationResult` フックによるインターセプト・オーバーライドも可能。認証フロー・パラメータ収集・承認ワークフローへの応用が広がる。

#### 12. MCP `list_changed` 通知サポート
- **バージョン:** 2.1.0
- **カテゴリ:** MCP Integration
- **概要:** 再接続なしでMCPサーバーのツール・プロンプト・リソースを動的に更新可能。長時間のエージェントワークフローや権限に応じて動的にツールを出し分けるMCPサーバーとの相性が格段に向上。

#### 13. claude.ai MCPコネクターをClaude Codeで使用可能に
- **バージョン:** 2.1.46
- **カテゴリ:** MCP Integration
- **概要:** claude.aiで設定・管理しているMCPコネクターをClaude Codeからそのまま利用可能に。WebUIとCLIツール間でMCP設定を二重管理する手間がなくなり、エコシステム全体の一貫性が確立。
